# 03 — A Stickelberger-type firing criterion for the fold-tower dichotomy

## Statement

The small-image fold-tower method (behind the $q=5185$ and $q=62305$
GRH-tier kills, Appendix B) proceeds by a general dichotomy, quoted
verbatim (source: `turyn_theory/small_image_family.md` §3, in the
source-repo repo — kept behind, not copied into this repo):

> **Dichotomy.** If a Turyn pair exists then `gamma = chi(tau) ∈ O_L` is a
> Weil q-number with balanced pattern ideal [unconditional], and:
>
> 1. the push-down `N_{L/F}(gamma)` forces the folded pattern ideal
>    principal in `Cl(F)` ... [unconditional reduction; evaluating it needs
>    `Cl(F)` — the ONLY class-group input];
> 2. **firing condition**: if for every non-constant p'-pattern S no choice
>    of the other primes' fold weights ("rescue weights") makes the class
>    vanish, the p'-pattern is constant [tier of `Cl(F)`];
> 3. constant p'-pattern ⟹ `Stab((gamma)) ⊇ U(p')` ... [unconditional] ⟹ by
>    `lem:stabilizer-rigidity` the true multiplier image contains the
>    order-`w·|U|` subgroup [unconditional];
> 4. if the exact predicate is empty at every `w'` with `w|U| | w' | t-1`, no
>    pair exists [unconditional, `cor:full-torus-exact-char`].

Step 2 — the "firing condition" — is exactly where the open problem lives.
Quoted verbatim (§5, "Gap 1 — the Stickelberger gap"):

> Step 2 of the dichotomy cannot currently be stated as an arithmetic
> condition on (q, t, w, p'): both instances reduced to "the p'-side
> degree-one primes of `k_e(i)` share a nonzero image in a small cyclic
> quotient of `Cl^-` and the rescue set degenerates", but predicting either
> fact requires the class group per instance. The deliverable-shaped
> missing lemma: *an explicit Stickelberger/reflection description of the
> subgroup of `Cl^-(k_e(i))` generated by the classes of the degree-one
> primes above `p'`, computable from (t, e, p') alone.* Until it exists,
> every new instance costs one bnfinit (fine at disc ≤ ~1e23) and its
> certification (the unconditionality bottleneck).

## Why it matters

Every fold-tower kill so far ($q=5185$, $q=62305$) required computing a
specific class group `Cl(F)` by hand (`bnfinit`, GRH-conditional) and
checking, instance by instance, whether the rescue-class set degenerates.
There is no theorem that predicts this in advance from $(q,t,w,p')$ alone.
Without it, the method does not scale past the handful of orders someone
has actually run `bnfinit` on, and each new instance also needs its own
(currently GRH-conditional) certification.

## What is known

Both worked instances are documented with their exact rescue-set
computations (§4, "$q=62305$..."; the $q=5185$ instance in the companion
note `q5185_small_image.md`). Both used the same pattern: a fold field
`k_e(i)` of manageable discriminant, a class group computed by `bnfinit`
under GRH, and an unconditional analytic cross-check via `h^-` matching the
class number — "same pincer quality as 5185" for the $q=62305$ case.

## Suggested attack

The gap statement itself names the deliverable precisely: an explicit
Stickelberger/reflection description of the relevant subgroup of
`Cl^-(k_e(i))`, generated by the classes of the degree-one primes above
`p'`, computable from `(t, e, p')` alone, without running `bnfinit` per
instance. This is a class-field-theory problem (reflection theorems for
`Cl^-` of Kummer/cyclotomic-type fields), not a computational one — see
**04-wall-witnesses.md** for the two composite $q$ that need it most,
since a per-instance `bnfinit` is not even feasible there.
