# Audit log

Provenance record for correctness / adversarial audits of this capstone.
One entry per audit, **newest first**. Referenced by `STATUS.md` ("Provenance")
as the running record of the human-directed verification of the paper's claims.
Each audit is expected to append an entry; do not rewrite past entries.

**Per-environment coverage** (one row per theorem-like environment, 128 total as
of 2026-07-08 — see that date's entry for the removal that took it from 129)
is tracked in `AUDIT_LEDGER.md`, generated by `code/tools/build_audit_ledger.py`
and cross-validated against `paper/capstone.aux`. Per-environment verdicts live
in `code/audit/verdicts.json`; an audit that signs off individual statements
populates that file and regenerates the ledger.

## Entry format

- **Date** — ISO date.
- **Models** — mainline orchestrator/verifier + any subagents, each with its role.
  (Record the model *family*; subagent model is whatever the `Agent` call requested.)
- **Scope** — which claims / files were in scope.
- **Method** — how the audit ran; what "verified" means here; adversarial or not.
- **Findings** — ranked most-severe-first; each with file:line location and severity.
- **Fixes applied** — `file` : change, or "none".
- **Open / not done** — recommended-but-deferred items (human decision).

## 2026-07-08 — Complete per-statement adversarial audit (129/129 environments)

**Models.** Sonnet 5 orchestrator (count-partitioned dispatch, checkpointing, synthesis;
did no deep math itself). **13 independent Opus 4.8** (`claude-opus-4-8`, high effort)
instances, one per count-partitioned batch of 9–10 theorem-like environments (batch 1
spans 1.1–3.7, confirming partitioning is by count, not by section — §4 alone holds 103
of the 129 environments). **1 Fable 5** (`claude-fable-5`) budget-limited second-opinion
pass over the 5 labels the Opus batches left UNVERIFIED. Batch 1 was run alone first as
a calibration check on depth before the remaining 12 were dispatched in parallel.

**Scope.** Every one of the 129 theorem-like environments in `paper/capstone.tex` —
statement AND proof, independently — closing the gap left by all prior audits, which
recorded only aggregate counts (e.g. "129/129", "95/101 SOUND") with no per-statement
ledger preserved in the repo. Before this audit, 111 of 129 environments were
"aggregate-only" coverage (see `AUDIT_LEDGER.md`), meaning no individual record existed
that a future audit could check against. This closes that gap for all 129.

**Method.** Adversarial and independent: each Opus batch was instructed to restate the
claim, reconstruct the proof step-by-step from scratch (not "checked, OK"), re-run any
finite/computational certificate the proof relies on via shipped `code/` (through
`~/.venvs/claude/bin/python` only — a global hook blocks bare `python3`/`pip`), and reach
one of SOUND / GAP / OVERCLAIM / UNVERIFIED per `code/audit/README.md`'s vocabulary. A
private "source-repo"/hadamard-search codebase was off-limits throughout: a proof step
not actually present or derivable in the capstone itself, and not reproducible from
shipped `code/`, was required to verdict UNVERIFIED, never SOUND, regardless of whether
the underlying math is well-known to be fine. Prior "aggregate-only" coverage notes were
explicitly not treated as evidence — every statement was audited from zero. Progress was
checkpointed after every batch via `code/tools/merge_verdicts.py` (atomic, resumable) so
a mid-run interruption would lose at most one batch. Several agents independently wrote
from-scratch verification code (brute-force enumerators, MITM solvers, ring-homomorphism
checks, cyclic-autocorrelation reconstructions) rather than trusting the shipped
scanners' output at face value — e.g. one batch built its own F_{q^2}-character harness
for the entire Appendix A construction, since no shipped script covers it.

**Findings, ranked most-severe-first.**

1. **GAP — `prop:fieldfree`** (`capstone.tex:3703-3716`, esp. the inference at
   3714-3715). The stated biconditional's reverse direction is unjustified: it equates
   a ray-class-group equality `cl_B(σI)=[μ⁻¹r]_B` — which is unit-blind (the exact
   sequence `O_L^× → (O_L/M_B)^× → Cl_B` kills unit residues) — with the literal
   congruence condition (C), which requires a generator of the *specific* norm-q
   (not merely "some") that hits the right residue. Norm-q generators of a fixed ideal
   differ only by `μ_4` (`lem:idealunit`), but the witnessing generator in the ray-class
   statement can differ by any unit. For `m` with positive unit rank (e.g. the section's
   own frontier case `m=31`, unit rank 14), a non-torsion unit can make the RHS hold
   while (C) genuinely fails. Recommended fix (not applied — report-only audit): restate
   as necessity-only (C ⟹ ray equality), or work modulo the unit subgroup the way
   `thm:absorbedC` attempts.

2. **UNVERIFIED, unresolved disagreement between the two audit passes —
   `thm:absorbedC`** (`capstone.tex:3736-3775`). The Opus batch verdicted UNVERIFIED,
   reasoning that Λ_B, read literally as generated by `{z/c(z) : z∈G_B}`, equals the
   full norm-one subgroup `N¹(G_B)`, which would trivialize the criterion (every source
   would collapse to one coset) — apparently contradicting the paper's own WIPEOUT rows
   in the shipped cache. The Fable second-opinion pass **specifically refutes that
   rationale**: it shows by hand that `O_L=O_K[i]` (disc(K_m) is odd, coprime to
   disc(Q(i))=-4), so `c` acts as the *identity* on `O_L/2O_L`, meaning Λ_B's 2-part has
   order only 2, not the full norm-one group — no trivialization, and the WIPEOUT rows
   are unaffected. However, the Fable pass then argues for a **different** defect in the
   same theorem: the reverse-iff direction (3752-3755, esp. "the generators of Λ_B are
   precisely these absorbed choices" at 3769-3770) is unjustified because `lem:idealunit`
   only licenses a `μ_4` generator ambiguity, not the full `z/c(z)` twist, so a
   coset-intersection witness need not correspond to any realizable Weil generator — and
   recommends GAP instead of UNVERIFIED, while conceding UNVERIFIED is defensible
   pending the unshipped confirming script (`condition_c_absorption.py`, disclosed
   absent from `code/` at capstone.tex:3795-3799). Per audit protocol this disagreement
   was **not** auto-resolved: `merge_verdicts.py` was run without `--force` and
   correctly SKIPped the differing verdict (confirmed: `SKIP (differs, use --force):
   thm:absorbedC: UNVERIFIED -> GAP`), so `verdicts.json` retains the Opus UNVERIFIED
   verdict. **This needs human adjudication** — both readings agree something about the
   reverse direction is at least questionable and neither claims the WIPEOUT-based
   closures in §6 are wrong (both agree those only need the trivially-sound forward
   direction), but they disagree on the precise technical characterization of Λ_B and
   on whether the right label is UNVERIFIED or GAP.

3. **UNVERIFIED, honest and properly disclosed (not defects) — 4 environments.**
   `prop:q5185-f4-cut` and `prop:q5185-closure` (class group `Cl(F_4)` via GRH/`bnfinit`,
   off-limits source-repo PARI, not shipped); `prop:q62305-closure` (same pattern,
   `Cl(k_3(i))=C_5799`); `prop:small-factor-sieve` (conditioned on hypothesis `EQ_3(θ)`
   plus imported beta-sieve/Mertens constants from Friedlander–Iwaniec and
   Halberstam–Richert, not reproducible from shipped code). Both the Opus batches and
   the independent Fable pass confirm: in every case the paper's own text, `STATUS.md`
   (tiers T1/T2), and (for the sieve) `rem:lane-density-status` explicitly disclose the
   external/conditional input and do not overclaim past it. All arithmetic and logic
   *downstream* of the external input was independently reconstructed and verified in
   each case (e.g. the full stabilizer-rigidity → amplified-image → two-square/row-sum
   dichotomy chain for q=5185 and q=62305).

4. **Cosmetic / non-load-bearing prose issues found along the way** (all verdicted
   SOUND, since none affects a proof's validity — listed for optional editorial cleanup):
   - `lem:coherence` / `lem:composite-coherence` (`capstone.tex:293`): "all values
     exceed −φ(e)/(P−1)" is false at the boundary for prime e (equality, not exceedance);
     downstream uses retain strict slack, so no conclusion is affected. Suggested fix:
     "exceed" → "are at least".
   - `rem:chen-input` (`capstone.tex:2975-2976`): misattributes a `P_2` result to
     Richert's degree-one specialization; Richert's `f(p)=P_{2 deg f+1}` gives `P_3` at
     degree one, and the `P_2` result for a linear form is Chen's improvement. The
     underlying fact (6p−1=P_2) is true; only the attribution/exponent is off.
   - `prop:fold-dichotomy` (`capstone.tex:1369`): "the 61-type pinned parts are always
     stable" imports q=5185-specific language into a general proposition; means "any
     prime whose pattern space is a single point," not a logical gap.
   - `thm:4373`: the `t=73` case omits the explicit "≡1 mod 4" stated for the `t=43`
     case, though both primes (5, 29) satisfy it and it is needed for rigid(ii).
   - `lem:orth` (`capstone.tex:4467`): the standing hypothesis "χ nontrivial (m≥2)" is
     used but not stated; the section's scope (Turyn case m≥2) supplies it.
   - `rem:small-image-conditional` (`capstone.tex:1309`): "the unique one divisible by
     5+6i" is only unique after orientation/sign is pinned; the accompanying row-sum
     uniqueness claim is independently rigorous, so the conclusion stands regardless.
   - `cor:h3-support-sum-prime`: the surrounding prose census ("19 nodes at q≤2000, 71
     at q≤10000") was not exactly reproduced (independent recount: 20/75); the gap is
     presumably nodes already closed at an earlier hierarchy stage. Not part of the
     corollary's logical claim.
   - `cor:h3-support-sum-gcd`: correct as written but strictly weaker than
     `prop:h3-support-augmentation`/`cor:h3-support-sum-prime`; a candidate demotion to
     Remark is editorial, not a correctness fix (consistent with the prior audit's note).
   - `cor:c27-auto-box`: bound `h≥57` is correct but not sharp (`h≥55` suffices with a
     parity refinement) — already identified and explicitly HELD as-is in a prior audit
     pass; reconfirmed here as loose-but-correct, not a defect.

**Tally.** 123 SOUND · 5 UNVERIFIED (4 honest external-input limitations + 1 with an
unresolved cross-model disagreement, see Finding 2) · 1 GAP (`prop:fieldfree`) ·
0 OVERCLAIM. Full per-label detail in `code/audit/verdicts.json` and `AUDIT_LEDGER.md`
(129 rows, 129 verdicts).

**Fixes applied.** None — this is a report-only audit per its binding rules (no edits to
`paper/capstone.tex` or the companion). All recommended fixes above are for human review.
